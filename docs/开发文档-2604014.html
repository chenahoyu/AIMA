项目概览

AI‑MA 是一个基于 Streamlit 的 Web 控制台：用户输入自然语言（例如“把 101 到 105 号灯调成红色，亮度 50%”），系统调用 LLM 生成结构化 JSON，再由 UI 组装为 grandMA3 OSC 命令并下发。

主入口
app.py
核心模块
src/ai / src/osc / src/utils
本地持久化
src/.conversations / src/.api_cache

技术栈

依赖版本以 requirements.txt 为准（下列为 v0.5 关键技术栈摘要）。

Python 3.10+（建议） Streamlit 1.28.0 Pydantic v2 OpenAI SDK 1.6.x（兼容接口） httpx（含 SOCKS 代理） python-osc ChromaDB（预留知识库/向量检索） pytest / black / flake8
前端形态：Streamlit 单页应用，UI 由 app.py 直接渲染（无独立 React/Vue）。
通信形态：LLM 通过 HTTP（OpenAI 兼容）；MA3 通过 OSC/UDP（地址固定 /cmd）。

组件 / 模块功能

  • app.py（Streamlit UI + 编排）
    • 侧边栏：对话管理、OSC 设置、API 设置、网络/代理策略、自动化开关、使用说明。
    • 主区：状态条（MA 连接 + API 状态 + Model）、聊天记录、输入框、待发送确认与自动发送。
    • 本地缓存：API Key、API Base URL、模型 ID、UI 开关、语言等（通过 src/utils/api_key_cache.py）。
  • src/ai/translator.py（语义翻译器）
    • 使用 OpenAI 兼容 chat.completions.create 调用模型，强约束输出 纯 JSON（并使用 actions 列表结构）。
    • 包含 JSON 提取/修复兜底（例如模型输出夹杂文本）。
    • 支持代理与网络策略：trust_env、proxy_url（含 socks5）。
  • src/osc/osc_client.py（OSC 客户端）
    • 固定向 OSC 地址 /cmd 发送字符串（grandMA3 侧解析命令）。
    • 发送前清洗命令：禁止包含 /cmd 前缀、去掉外层引号。
    • 多条命令之间加入微小延迟（默认 50ms）提高稳定性。
  • src/utils/conversation_manager.py（对话管理）
    • 创建/列举/删除/重命名对话；消息追加；持久化到 src/.conversations/conversations.json。
    • 按更新时间倒序列出，便于 UI 显示“最近会话”。
  • src/utils/api_key_cache.py（配置缓存）
    • 将 key/value 缓存到 src/.api_cache/api_keys.json；写入后设置文件权限 600。
    • 注意：当前为 Base64 编码，不等价于安全加密。
  • config.py（默认配置）
    • 使用 Pydantic Settings 统一管理：MA3 OSC host/port、OpenAI base_url、model_id、数据目录等。

开发要点（重要）

  • Streamlit 的 st.session_state 是核心状态容器：UI 交互会触发 rerun，所有“跨 rerun 状态”需要放在 session_state。
  • 对话选择器与 rerun：下拉框（selectbox）一旦实例化，直接改同名 key 容易触发异常；推荐“pending 值 → 下次 rerun 写回”的方式。
  • 网络/代理策略：API 调用受系统代理环境变量影响；项目支持 直连/系统代理/环境 SOCKS5/自定义代理，并提供“API 可用性验证”用于快速诊断。
  • OSC 命令必须清洗：发送给 grandMA3 的 value 不能带 /cmd 前缀；字符串外层引号需要剥离。
  • 多命令发送节流：多条 OSC 连发要加 delay（避免 MA3 解析失败）。
  • 缓存落盘的边界：本地缓存适合个人使用；若上云或多人共享，建议改为环境变量或 Secret Manager。
  • 多语言 i18n：UI 文案采用 key→翻译表的方式集中管理，避免散落硬编码；切换语言应全局 rerun 生效。
建议约定：UI 文案、状态提示、错误信息尽量统一走翻译/字典层；避免“部分英文、部分中文、部分硬编码”的不可维护状态。

启动与调试

推荐在 backup/v0.5/ 下使用虚拟环境启动。

# 1) 创建虚拟环境（如未创建）
python3 -m venv venv

# 2) 安装依赖
./venv/bin/python -m pip install -r requirements.txt

# 3) 启动（本地开发）
./venv/bin/python -m streamlit run app.py --server.address 127.0.0.1 --server.port 8502
常见问题：
  • 页面打不开：确认 Streamlit 进程是否启动、端口是否被占用、是否使用了正确的 Python（不要用系统 python 指向 Python2）。
  • API 连接失败：优先在 UI 中点击“验证 API 可用性”，并切换网络方式（直连/系统代理/SOCKS/自定义）。

安全注意事项

  • 不要提交真实的 .env：只保留 .env.example 作为示例。
  • API Key 不建议落盘：当前实现使用 Base64 编码，属于“可读性处理”，不是加密；若部署公网建议改为 Secret 管理并增加访问控制。
  • 公网部署必须做鉴权：建议通过 Nginx/Caddy 增加 HTTPS、Basic Auth、IP 白名单等。
