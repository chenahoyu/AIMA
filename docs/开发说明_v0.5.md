# AI-MA v0.5 开发说明（工程交接 / 二次开发）

本文档面向接手开发/部署的工程师，目标是让项目可以在**新设备/云服务器**上独立启动与维护。

> 版本标识：v0.5  
> 备份目录：`backup/v0.5/`

## 1. 项目用途概览

AI‑MA 是一个基于 Streamlit 的 Web 控制台，用自然语言生成 grandMA3 的指令，并通过 OSC（UDP）下发到 grandMA3。

- **UI**：`app.py`（Streamlit）
- **AI 翻译**：`src/ai/translator.py`（OpenAI 兼容 Chat Completions）
- **OSC 通信**：`src/osc/osc_client.py`（向 grandMA3 发送 `/cmd`）
- **对话与本地缓存**：
  - `src/utils/conversation_manager.py` → `src/.conversations/conversations.json`
  - `src/utils/api_key_cache.py` → `src/.api_cache/api_keys.json`（会缓存密钥，注意安全）

## 2. 目录结构（v0.5）

关键目录/文件：

- **`app.py`**：主入口（Streamlit UI + 业务编排）
- **`scripts/launcher.py`**：命令行启动器（语言选择 → 输入 IP/端口 → 环境自检 → 一键启动 Streamlit）
- **`docs/开发文档.html`**：静态 HTML 版开发文档（适合离线查看/交付）
- **`src/`**：
  - `src/ai/models.py`：Pydantic 数据结构（`GMA3Command`、`CommandAction`）
  - `src/ai/translator.py`：LLM 调用与 JSON 解析修复
  - `src/osc/osc_client.py`：OSC 客户端（向 `/cmd` 发送字符串）
  - `src/utils/api_key_cache.py`：API Key 缓存（Base64，非强加密）
  - `src/utils/conversation_manager.py`：对话存储/列表/删除
- **`.streamlit/config.toml`**：Streamlit 主题配置（强制暗黑主题）
- **`requirements.txt`**：依赖清单
- **`.env.example`**：环境变量示例（不要把真实 `.env` 提交/备份）
- **`data/`**：项目数据目录（知识库/fixtures/presets/sequences 等）

## 3. 技术栈与版本要求

- **Python**：建议 3.9+（本机 venv 为 3.9；如上云建议 3.10+）
- **Web 框架**：Streamlit 1.28.x
- **AI SDK**：OpenAI Python SDK 1.6.x（OpenAI 兼容接口）
- **HTTP 客户端**：httpx（用于验证 API、以及作为 OpenAI SDK 底层 client）
- **SOCKS 代理支持**：
  - `httpcore[socks]` + `socksio`
- **OSC**：python-osc
- **数据模型**：pydantic v2

## 4. 启动、停止、端口

### 4.1 本地开发启动

在项目目录（本备份根目录 `backup/v0.5/`）执行：

```bash
python -m venv venv
./venv/bin/python -m pip install -r requirements.txt
./venv/bin/python -m streamlit run app.py --server.port 8502
```

打开：`http://127.0.0.1:8502`

> 说明：若你本机项目的虚拟环境不在 `backup/v0.5/venv`，可以直接使用仓库根目录的 venv（例如 `AI-MA/venv`）来运行本备份版本的代码。

### 4.1.1 使用命令行启动器（推荐）

启动器支持：**先选语言（中文/English/日本語）→ 输入 IP/端口 → 自动自检依赖 → 一键启动网页服务**。

推荐用项目 venv 运行（避免系统 Python 缺依赖）：

```bash
/Users/chenhaoyu/AI-MA/venv/bin/python backup/v0.5/scripts/launcher.py
```

### 4.2 停止服务

最简单方式：在启动终端 `Ctrl+C`。

若需要按端口停止（macOS/Linux）：

```bash
lsof -nP -iTCP:8502 -sTCP:LISTEN -t | xargs kill
```

## 5. 核心功能说明（组件级）

### 5.1 UI（`app.py`）

主要功能模块：

- **对话管理**：创建/切换/删除对话、清空当前对话消息  
  - 对话下拉框 key 为 `conversation_selector`
  - v0.5 修复了 “删除对话后修改 session_state 导致异常” 的问题：用 `conversation_selector_pending` 在下一次 rerun 前更新选择器

- **OSC 设置**：配置 grandMA3 的 `host:port`，可点击“重新连接”

- **API 设置**：
  - 配置 `API Key`、`API Base URL`、`model_id`
  - **API Base URL 默认不明文常驻显示**：侧边栏默认仅显示“已缓存/未设置”，需要时点击“编辑”才展开输入框，保存后可收起
  - **API 可用性验证**：发送最小化请求，返回明确的 HTTP 状态/错误原因
  - **网络方式（影响 API 调用）**：
    - 直连（不走代理）
    - 系统代理（HTTP_PROXY/HTTPS_PROXY/ALL_PROXY）
    - 环境 SOCKS5（SOCKS5_PROXY / SOCKS_PROXY）
    - 自定义代理（http(s):// 或 socks5://）

- **全局语言切换（i18n）**：
  - 支持中文 / English / 日本語 的全局切换
  - UI 文案集中在 `app.py` 的翻译表中，通过 `t(key)` 统一渲染
  - 语言选择会缓存到本地（`ui_lang`），下次启动自动恢复

### 5.2 AI 翻译器（`src/ai/translator.py`）

- 调用 OpenAI 兼容的 `chat.completions.create(...)`
- system prompt 强约束输出 **纯 JSON**，并使用 `actions` 列表结构
- 对常见 JSON 格式问题进行修复/兜底提取
- 支持自定义网络策略：
  - `trust_env=True/False`（是否使用系统代理环境变量）
  - `proxy_url`（显式代理，支持 socks5://）

### 5.3 OSC 客户端（`src/osc/osc_client.py`）

- 发送地址固定为 `/cmd`
- 发送内容为“清洗后的命令字符串”（禁止 `/cmd` 前缀、去外层引号）
- 多条命令发送时带微小延迟（默认 50ms）提高 grandMA3 解析稳定性

### 5.4 本地数据与缓存

- **对话**：`src/.conversations/conversations.json`
- **API Key 缓存**：`src/.api_cache/api_keys.json`
  - 注意：当前是 Base64 编码，不等同于安全加密
  - 上云/多人协作建议改为：环境变量/外部 Secret Manager，不落盘或加密落盘

## 6. 云服务器部署要点（不封装）

### 6.1 服务器准备

- 安装 Python（建议 3.10+）与 pip
- 开放端口（例如 8502）到安全组/防火墙
- 确保服务器能访问你配置的 AI API 域名（必要时配置代理）

### 6.2 建议的启动命令

```bash
./venv/bin/python -m streamlit run app.py \
  --server.port 8502 \
  --server.address 0.0.0.0 \
  --server.headless true
```

### 6.3 反向代理（可选）

生产建议用 Nginx/Caddy 做 HTTPS 与访问控制（Basic Auth / IP 白名单）。

### 6.4 代理/DNS 常见坑

如果出现 “Connection error / DNS not known / ProxyError 403”：

- 先在 UI 里切换 **网络方式** 并点 **验证 API 可用性**
- 若系统 HTTP 代理返回 403，说明代理规则拦截 CONNECT，需要放行该域名
- 若直连 DNS 失败，说明服务器/进程环境无法解析外网域名（需要修 DNS 或走代理）

## 7. 安全注意事项

- **不要把真实 `.env` 进入备份/提交**（本 v0.5 备份仅保留 `.env.example`）
- **API Key 不建议落盘**：如需缓存，至少限制文件权限（600）并考虑加密
- 若部署到公网，务必做登录/鉴权/访问控制

## 8. 快速自检清单

- 能打开 Streamlit 页面
- “API 设置”里点 **验证 API 可用性** 能得到明确 HTTP 状态
- 能在“OSC 设置”正确配置 MA3 host/port
- 发送指令后能生成 OSC 命令（即使不连 MA3 也不应崩溃）
